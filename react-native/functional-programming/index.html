함수형 프로그래밍(Functional Programming)

2019-06-07

리액트 네이티브(RN - React Native)에서 함수형 프로그래밍(Functional Programming)을 사용해 봅시다.

  1. Home
  2. react-native
  3. 함수형 프로그래밍(Functional Programming)

개요

“리액트 훅(React Hooks)이 리액트의 새로운 패러다임이다!”라는 말을 듣고, ‘리액트 훅(React Hooks) 뭐지?’, ‘한번 공부해봐야지’하며 생각만 하다가 나온지는 한참됐는데 지금까지 공부하지 못 했습니다. 그러다가, 새로운 프로젝트를 시작하는 타이밍이여서, 리액트 훅(React Hooks)을 한번 시도해봐야겠다고 생각했습니다. 그래서 리액트 훅(React Hooks)을 공부하다 보니, 함수형 프로그래밍(Functional Programming)을 해야 하는 거 같더군요. 저는 C++, java와 같은 객체 지향 프로그래밍(OOP - Object Oriented Programming)으로 입문하다 보니 클래스 형식의 리액트(React)가 너무 좋았습니다. 뭔가 정형화된거 같고, 룰이 있고 틀이 잡혀있는거 같아서요. 그래도 사람들이 리액트의 새로운 패러다임이라고 불릴 정도로 리액트 훅(React Hooks)에 열광하는 걸 봐서는 역시 경험해볼 필요가 있다고 생각이 들었습니다.

이번 블로그에서는 간단하게 리액트 네이티브(RN - React Native)에서 함수형 프로그래밍(Fucntional Programming)을 하는 방법에 대해서 살펴보도록 하겠습니다.

프로젝트 생성

아래에 명령어로 리액트 네이티브(RN - React Native) 프로젝트를 생성합니다.

react-native init FunctionalReact

그리고 아래에 명령어를 통해 프로젝트를 실행합니다.

react-native run-ios
# react-native run-ios

저는 보통 package.json의 아래의 스크립트를 추가하여 사용하고 있습니다.

...
"scripts": {
    ...
    "ios": "react-native run-ios",
    "android": "react-native run-android"
},
...

아래의 명령어로 위에서 실행한 것처럼 프로젝트를 실행할 수 있습니다.

npm run ios
# npm run android

그러명 아래와 같이 프로젝트가 잘 실행되는 것을 확인할 수 있습니다.

함수형 프로그래밍

그럼 이제 현재의 소스를 함수형 프로그래밍 스타일로 변경해 보도록 하겠습니다. 현재 소스는 아래와 같습니다.

import React, {Component} from 'react';
import {Platform, StyleSheet, Text, View} from 'react-native';

const instructions = Platform.select({
  ios: 'Press Cmd+R to reload,\n' + 'Cmd+D or shake for dev menu',
  android:
    'Double tap R on your keyboard to reload,\n' +
    'Shake or press menu button for dev menu',
});

type Props = {};
export default class App extends Component<Props> {
  render() {
    return (
      <View style={styles.container}>
        <Text style={styles.welcome}>Welcome to React Native!</Text>
        <Text style={styles.instructions}>To get started, edit App.js</Text>
        <Text style={styles.instructions}>{instructions}</Text>
      </View>
    );
  }
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#F5FCFF',
  },
  welcome: {
    fontSize: 20,
    textAlign: 'center',
    margin: 10,
  },
  instructions: {
    textAlign: 'center',
    color: '#333333',
    marginBottom: 5,
  },
});

이제 이 소스코드를 함수형 프로그래밍(Functional Programming) 방식으로 바꿔보도록 하겠습니다.

import { Platform, StyleSheet, Text, View } from 'react-native';

    'Shake or press menu button for dev menu',
});

export default () => {
  return (
    <View style={styles.container}>
      <Text style={styles.welcome}>Welcome to React Native!</Text>
      <Text style={styles.instructions}>To get started, edit App.js</Text>
      <Text style={styles.instructions}>{instructions}</Text>

  );
};

    fontSize: 20,
    textAlign: 'center',

    textAlign: 'center',

});

음…뭔가 심플해진거 같긴합니다.

타입스크립트(Typescript) 적용

저는 보통 리액트 네이티브(RN - React Native) 프로젝트를 진행할 때, 타입스크립트(typescript)를 사용하고 있습니다. 자세한 내용은 아래의 블로그를 참고하시기 바랍니다.

  • typescript

이전 스타일에서 타입스크립트(typescript)를 적용해 보겠습니다.

import { Platform, StyleSheet, Text, View } from 'react-native';

const instructions = Platform.select({

  android:
    'Double tap R on your keyboard to reload,\n' +
    'Shake or press menu button for dev menu',
});

interface State {}
interface Props {}

export default class App extends React.Component<Props, State> {
  render() {
    return (
      <View style={styles.container}>
        <Text style={styles.welcome}>Welcome to React Native!</Text>
        <Text style={styles.instructions}>To get started, edit App.js</Text>
        <Text style={styles.instructions}>{instructions}</Text>
      </View>
    );
  }
}

  container: {
    flex: 1,
