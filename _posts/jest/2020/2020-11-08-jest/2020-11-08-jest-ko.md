## 개요

이번 블로그 포스트에서는 JavaScript 테스트 프레임워크인 `Jest`에 대해서 설명하려고 합니다.

## Jest

Jest란 Facebook에서 개발 관리하는 JavaScript 테스트 프레임워크로 심플함에 집중한 프레임워크입니다. JavaScript 테스트 프레임워크이므로 Typescript, Node, React, Angular, Vue 등에서 사용할 수 있습니다.

## 장점

Jest는 다음과 같은 장점을 가지고 있습니다.

### 제로 설정

많은 테스트 프레임워크들이 테스트를 하기 위해서 많은 설정을 해야합니다. Jest는 이런 설정때문에 Test를 쉽게 시작하지 못하고, Test에 집중하지 못하는 단점을 보완하고자 했습니다.

그래서 Jest는 제로 설정(Zero Config)을 내세워 아무 설정없이 Test를 쉽게 시작할 수 있게 하고 좀 더 Test에 집중하도록 하였습니다.

### 스냅샷

테스트를 하다보면 큰 오브젝트 결과값을 일일이 확인하기가 어렵습니다. Jest는 스냅샷(Snapshots) 기능을 제공하여, 일일이 값을 확인하기 어려운 큰 오브젝트 결과값을 조금 더 쉽게 테스트할 수 있도록 하였습니다.

### isolated

테스트 코드는 완전히 분리되어 있으며(isolated), 이렇게 분리된 테스트는 동시 실행이 가능합니다. 따라서 Jest는 동시에 테스트 코드를 실행하므로 빠른 성능을 제공합니다.

### API

Jest는 쉽고 간단하게 테스트할 수 있는 뛰어난 API를 제공합니다. 또한 `--coverage` 옵션을 통해 코드 커버리지(Code Coverage)를 간단하게 확인할 수 있습니다.

### 모의 객체

Jest는 쉽게 모의 객체(Mocking)를 생성할 수 있습니다. 이를 통해 테스트 범위를 벗어나는 객체들을 간단하게 모의 객체로 만듬으로써 좀 더 테스트에 집중할 수 있습니다.

## 철학

프레임워크를 사용할 때, 프레임워크의 철학을 이해하면 프레임워크를 사용할 때 많은 도움이 됩니다. 프레임워크를 사용하다 보면 왜 이런 API가 존재하는지 왜 이렇게 사용해야하는지 등이 이해가 안갈 떄가 있습니다. 하지만 프레임워크의 철학을 이해하면 이런 내용들이 조금 더 쉽게 이해가 됩니다.

다음은 Jest 프레임워크의 철학입니다.

> Jest is a JavaScript testing framework designed to ensure correctness of any JavaScript codebase. It allows you to write tests with an approachable, familiar and feature-rich API that gives you results quickly.
>
> Jest is well-documented, requires little configuration and can be extended to match your requirements.
>
> Jest makes testing delightful.
>
> - Jest Core Team

번역해 보면 다음과 같습니다.

> Jest는 JavaScript 코드베이스의 정확성을 보장하도록 설계된 JavaScript 테스트 프레임워크입니다. Jest는 여러분에게 접근성이 좋고 친숙하며 기능이 풍부한 API를 통해 신속하게 결과를 얻을 수 있도록 합니다.
>
> Jest는 문서화가 잘되어 있고 설정이 거의 필요없으며 여러분의 요구 사항에 맞게 확장할 수 있습니다.
>
> Jest는 테스트를 즐겁게 만들어 줍니다.
>
> - Jest Core Team
