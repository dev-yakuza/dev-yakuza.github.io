## 개요

Rails는 웹 애플리케이션 개발 프레임워크이므로 URL과 Controller를 연결해 주는 기능을 제공합니다. 이 URL 관리 기능을 Routes라고 하며 Rails에서는 `config/routes.rb` 파일에서 관리합니다.

이번 블로그 포스트에서는 Rails에서 URL을 관리하기 위한 Routes에 관해 살펴보겠습니다.

## get/post

Rails에서 HTTP의 `GET/POST` 메소드에 상응하는 URL은 다음과 같이 `config/routes.rb` 파일에서 사용할 수 있다.

```ruby
MyApp::Application.routes.draw do
  get 'books/:id', to: 'books#show'
  # post 'books/:id', to: 'books#show'
  # get 'books/:id', action: :show, controller: 'users'
end
```

또한 `as` 옵션을 사용하여 URL 생성시 사용할 이름을 지정할 수 있다.

```ruby
MyApp::Application.routes.draw do
  get 'books/:id', to: 'books#show', as 'book'
  # post 'books/:id', to: 'books#show', as 'book'
end

# book_path(12)
```

## resources

Rails에서는 `resources`라는 메서드를 제공하며 다음과 같이 `config/routes.rb` 파일에서 사용할 수 있다.

```ruby
MyApp::Application.routes.draw do
  resources :books
end
```

이렇게 추가된 `resources`는 다음과 같은 URL을 자동 생성하고 매칭한다.

| URL | 액션 | HTTP 메서드 | 설명 |
| --- | --- | --- | --- |
| /books(:format) | books/index | GET | 목록 화면 표시 |
| /books(:format) | books/show | GET | 개별 상세 화면 표시 |
| /books/new(:format) | books/new | GET | 새로운 데이터 등록 화면 표시 |
| /books(:format) | books/create | POST | 새로운 데이터 등록 화면의 입력을 받아 데이터 등록 처리 |
| /books/:id/:edit(:format) | edit | GET | 수정 화면 표시 |
| /books/:id(:format) | books/update | PATCH/PUT | 수정 화면의 입력을 받아 수정 처리 |
| /books/:id(:format) | books/destroy | DELETE | 목록 화면에서 지정된 데이터 제거 처리 |

resources에서는 `only` 옵션을 사용하여 필요한 URL만 사용할 수 있도록 지정할 수 있다.

```ruby
MyApp::Application.routes.draw do
  resources :books, only: [:index, :show]
end
```

이렇게 `only` 옵션을 지정하면 다음과 같이 일부 URL만 생성된다.

| URL | 액션 | HTTP 메서드 |
| --- | --- | --- |
| /books(:format) | books/index | GET |
| /books(:format) | books/show | GET |

## namespace

Rails에서는 `namespace`를 사용하여 URL에 prefix를 설정할 수 있습니다. `config/routes.rb` 파일에서 다음과 같이 사용할 수 있습니다.

```ruby
MyApp::Application.routes.draw do
  namespace :admin do
    resources :books
  end
end
```

이렇게 사용된 `namespace`는 다음과 같이 URL에 적용됩니다.

| URL | 액션 | HTTP 메서드 |
| --- | --- | --- | --- |
| /admin/books(:format) | admin/books/index | GET |
| /admin/books(:format) | admin/books/show | GET |
| /admin/books/new(:format) | admin/books/new | GET |
| /admin/books(:format) | admin/books/create | POST |
| /admin/books/:id/:edit(:format) | admin/books/edit | GET |
| /admin/books/:id(:format) | admin/books/update | PATCH/PUT |
| /admin/books/:id(:format) | admin/books/destroy | DELETE |

## scope

Rails에는 `namespace`와 비슷한 역할을 하는 `scope`가 있습니다. `scope`는 다음과 같이 사용할 수 있습니다.

```ruby
MyApp::Application.routes.draw do
  scope :admin do
    resources :books
  end
end
```

이렇게 사용된 `scope`는 다음과 같이 URL에 적용됩니다.

| URL | 액션 | HTTP 메서드 |
| --- | --- | --- | --- |
| /admin/books(:format) | books/index | GET |
| /admin/books(:format) | books/show | GET |
| /admin/books/new(:format) | books/new | GET |
| /admin/books(:format) | books/create | POST |
| /admin/books/:id/:edit(:format) | books/edit | GET |
| /admin/books/:id(:format) | books/update | PATCH/PUT |
| /admin/books/:id(:format) | books/destroy | DELETE |

### module 옵션

Rails의 scope는 `module`이라는 옵션을 가지고 있습니다. scope의 `module` 옵션은 다음과 같이 사용할 수 있습니다.

```ruby
MyApp::Application.routes.draw do
  scope module: 'admin' do
    resources :books
  end
end
```

이렇게 사용된 scope의 `module` 옵션은 다음과 같이 URL에 적용됩니다.

| URL | 액션 | HTTP 메서드 |
| --- | --- | --- | --- |
| /books(:format) | admin/books/index | GET |
| /books(:format) | admin/books/show | GET |
| /books/new(:format) | admin/books/new | GET |
| /books(:format) | admin/books/create | POST |
| /books/:id/:edit(:format) | admin/books/edit | GET |
| /books/:id(:format) | admin/books/update | PATCH/PUT |
| /books/:id(:format) | admin/books/destroy | DELETE |

### path 옵션

Rails의 scope는 `path`라는 옵션도 가지고 있습니다. scope의 `path` 옵션은 다음과 같이 사용할 수 있습니다.

```ruby
MyApp::Application.routes.draw do
  scope module: 'admin', path: 'foo' do
    resources :books
  end
end
```

이렇게 사용된 scope의 `module` 옵션은 다음과 같이 URL에 적용됩니다.

| URL | 액션 | HTTP 메서드 |
| --- | --- | --- | --- |
| /foo/books(:format) | admin/books/index | GET |
| /foo/books(:format) | admin/books/show | GET |
| /foo/books/new(:format) | admin/books/new | GET |
| /foo/books(:format) | admin/books/create | POST |
| /foo/books/:id/:edit(:format) | admin/books/edit | GET |
| /foo/books/:id(:format) | admin/books/update | PATCH/PUT |
| /foo/books/:id(:format) | admin/books/destroy | DELETE |

### as 옵션

Rails의 scope는 `as`라는 옵션도 가지고 있습니다. scope의 `as` 옵션은 다음과 같이 사용할 수 있습니다.

```ruby
MyApp::Application.routes.draw do
  scope module: 'admin', path: 'foo', as: 'cool' do
    resources :books
  end
end
```

이렇게 사용된 scope의 `as` 옵션은 다음과 같이 URL에 적용됩니다.

| URL | 액션 | HTTP 메서드 |
| --- | --- | --- | --- |
| /foo/books(:format) | admin/books/index | GET |
| /foo/books(:format) | admin/books/show | GET |
| /foo/books/new(:format) | admin/books/new | GET |
| /foo/books(:format) | admin/books/create | POST |
| /foo/books/:id/:edit(:format) | admin/books/edit | GET |
| /foo/books/:id(:format) | admin/books/update | PATCH/PUT |
| /foo/books/:id(:format) | admin/books/destroy | DELETE |

scope의 `as` 옵션은 URL에는 아무 변화를 가져오지 않는다. 단지 이 URL을 사용할 때 `cool_book`으로 참조하여 사용할 수 있도록 한다.
